A) Quick Start Guide - top

To install the complete script including the image uploader (as seen in the demo), all you have to do is upload all the files that came with this package to your webserver. Once you have done this, the image uploader page will appear on the index.php file. Example: http://www.yoursite.com/index.php

That's it! There's no reason to read any further unless you want to use the more advanced applications of this script. Everything after this will require some basic knowledge of HTML tags.

B) Uploading Images - top

To use the image uploader located in "index.php", first boot up the page (like in the example above), click on the "Browse" button to browse your computer for an image to be converted to ASCII characters.

All uploaded images will be automatically stored in the /uploads/ directory on your web server. To delete all of these images automatically, see Deleting Images.

C) Deleting Images - top

To automatically erase all images located in the /uploads/ directory, just run "cleanup.php" from wherever you uploaded the files to. Example: Type "http://www.yoursite.com/cleanup.php" into your browser. If the file has ran successfully, a message saying "All images deleted!" should show up.

D) Manually Converting Images - top

If you would like to convert images for yourself on your web pages without the uploader, this will require some minor knowledge of HTML. First, we need is to call on the "imgtoascii.min.js" file located in the /lib/ folder. To do this, copy and paste this code between your <head> tags:

<script type="text/javascript" src="lib/imgtoascii.min.js"></script>

Now we need to call on the script in our <img> tags by adding the attribute asciify="true" inside the tag, like so:

<img src="img/yourimage.jpg" asciify="true" />

You can control the resolution with the attribute asciiresolution=(low/medium/high) and the color with the attribute asciicolor=(true/false). So if we wanted a high resolution image with color, the code would be:

<img src="img/yourimage.jpg" asciify="true" asciiresolution="high" asciicolor="true" />

E) Known Issues - top

  • The only major known issue is with Internet Explorer. At the moment, the script does not work with IE and there is no way to fix that due to the lack of support for the getImageData function. I have included a simple PHP script in "index.php" that will detect if the browser is IE and ask them to switch to another browser. If compatibility with IE ever improves I will update the script to support IE.
  • There is a minor issue dealing with poorly compressed PNG files. The image will still be displayed but with the top few characters saying "undefined". I haven't found a workaround for this yet, but most properly formed PNG files will work just fine. The ones that don't will still be displayed minus a few pixels.

If you catch any errors, email me at kennedy@kennedyrose.com
